Atributos, Eventos y ngModel

Para entender esto un atributo en HTML son los modificadores que acompañan a las etiquetas como la palabra class cuando damos una clase a una etiqeta (class="seccionPadre"), pero los atributos en el contexto angular a menudo se usa la palabra "atributo" como "propiedad" pero no siempre los usa de esa manera. Hay que tener cuidado para no producir ningun mensaje de error a la hora de crear atributos ya que si no saldran mensajes de error. Algunos atributos los podemos ver en la suigiente direccion: https://www.w3schools.com/tags/tag_button.asp y los eventos aqui: https://www.w3schools.com/tags/ref_eventattributes.asp Estos atributo y eventos siempre van dentro de corchetes. Tambien veremos ngModel que es un atributo para formularios que sirve tanto de atributo como de evento para ello hay que importar en al module.ts FormsModule y con esto importamos el codigo que pertence a ngModel se puede utilizar de tres formas solo como atributo, solo como evento o con los dos. Si la asignamos como atributo (entre corchetes) al escribir un valor en el .ts se asignaria ese valor en la pantalla, pero si lo asigno como evento (entre parentesis) al escribir algo en el input se asiganria ese valor a la variable y si quremos usarlo para ambos como evento y como atributo sereia [(ngModel)] esto haria que se sincronize con el componente ahora veremos de una manera mas detallada como es todo esto que hemos hablado.




Asignandolo como evento y como atributo se guardara lo que escribamos en la variable de .ts y ademas saldra por pantalla, ademas si queremos ver que se a guardado en la variable abriendo la consola se vera el valor escrito. Esto solo sucedera cuando apretemos el boton asignar variable.
